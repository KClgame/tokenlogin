如何把你的 Mod 项目
上传到 GitHub

本教程专为你的 TokenLogin 项目设计，包含 Fabric Mod 项目的特殊注意事项。

推荐方式
GitHub Desktop（最简单）

图形化界面操作，几乎不会出错，强烈建议新手使用。

进阶方式
命令行（Git Bash / PowerShell）

更灵活，适合有一定 Git 基础的用户。

前置准备

1
GitHub 账号
注册并登录 github.com
2
安装 Git
下载地址：git-scm.com
（GitHub Desktop 会自动安装）
3
准备好项目文件夹
推荐使用：
tokenlogin-template-26.1.2 (1)
推荐新手

方法一：使用 GitHub Desktop

1
下载并安装 GitHub Desktop
前往官网下载
安装后使用 GitHub 账号登录
2
在 GitHub 创建新仓库
  1. 登录 GitHub 后点击右上角 + → New repository
  2. Repository name 建议填写：TokenLogin-26.1.2
  3. 选择 Public 或 Private
  4. 重要：不要勾选 “Add a README file”
  5. 点击 Create repository
3
克隆仓库到本地
在 GitHub Desktop 中点击 File → Clone repository，选择你刚创建的仓库，点击 Clone。
4
复制你的 Mod 文件（重要！）
操作建议：
  • 把 tokenlogin-template-26.1.2 (1) 里面的所有内容复制到刚克隆的仓库文件夹中
  • 不要复制 build、.gradle、run 这三个文件夹
5
创建 .gitignore 文件

在仓库根目录新建一个名为 .gitignore 的文件，内容如下：

# Gradle
.gradle/
build/
run/

# IDE
.idea/
*.iml
*.iws
*.ipr

# Minecraft
run/
.gradle/
build/
6
提交并推送
  1. 在 GitHub Desktop 左侧看到所有改动
  2. 下方填写提交信息，例如：Initial commit - TokenLogin 26.1.2
  3. 点击 Commit to main
  4. 再点击右上角的 Push origin
完成！你的代码已经上传到 GitHub 了

方法二：使用命令行（Git）

1. 进入项目目录并初始化
cd "C:\Users\Administrator\Desktop\homework\9\tokenlogin-template-26.1.2 (1)"
git init
2. 创建 .gitignore（同上）
3. 提交代码
git add .
git commit -m "Initial commit - TokenLogin 26.1.2"
4. 关联远程仓库并推送
git remote add origin https://github.com/KClgame/你的仓库名.git
git branch -M main
git push -u origin main

Minecraft Mod 项目特别提醒

以下文件夹绝对不要上传到 GitHub：
build/
.gradle/
run/
.idea/
这些文件夹体积巨大且包含大量临时文件，会让你的仓库变得非常臃肿。
